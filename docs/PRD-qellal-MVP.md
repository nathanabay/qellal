# Deep Research Request: Qellal — Ethiopian Tender Opportunity Platform

<context>
I'm building a tender opportunity aggregation platform for Ethiopia with some basic
technical knowledge but low coding experience. Companies and NGOs will view tender
opportunities collected from newspapers, newsletters, and official government portals.
The site will be FREE for the first 6 months to build a user base, then switch to paid
subscriptions (monthly, quarterly, yearly) using Chapa and Telebirr payment gateways.

**My Skills:** Beginner level — willing to learn any new tool that is optimal
**Learning Preference:** Optimal over familiar; I want beginner-friendly tools with
strong documentation and good AI-assistant support
**Timeline:** MVP live in 1–2 months
**Budget:** Free tools/tiers only during development and the free period
**Key Differentiator:** A notification system (email/SMS/Telegram) so subscribers never
miss relevant tenders — the main competitor (2merkato) lacks this
</context>

<instructions>

## Core Questions

1. What is the best beginner-friendly backend + database stack for a listings platform
   with user accounts, search/filtering, and notifications — buildable in 1–2 months
   on free tiers?
2. What free deployment/hosting options work reliably for users in Ethiopia
   (latency, uptime, free-tier limits)?
3. How do I cover BOTH web and mobile with minimal extra work — responsive web app,
   PWA, or cross-platform framework (Flutter/React Native)? Which suits a beginner?
4. How do Chapa and Telebirr integrations work technically? Compare fees, documentation
   quality, sandbox/test environments, subscription/recurring-payment support, and
   which no-code/low-code or SDK options exist.
5. What are beginner-friendly ways to scrape tender notices from newsletters and
   official Ethiopian portals (tools, legality, reliability), combined with a simple
   admin panel for manual data entry by employees?
6. What is the cheapest reliable way to send notifications in Ethiopia — email, SMS
   (local gateway costs), Telegram bot — and how do I let users subscribe to
   categories/keywords?

## Research Areas

- Market validation: 2merkato and other Ethiopian tender platforms — features,
  pricing, user complaints, gaps
- Realistic subscription pricing for the Ethiopian market (what businesses will pay)
- Legal/copyright considerations of republishing tender notices in Ethiopia
- Technical approach recommendations matched to a beginner
- AI coding tools (e.g., Claude Code, Cursor, v0, Bolt) to accelerate development
- Learning resources for the recommended stack
- No-code vs low-code vs full-code trade-offs for this specific product

## Specific Focus

- Implementation complexity of each approach for a beginner
- Time to MVP with different tools (must fit 1–2 months)
- Running costs now (free) and after the free period ends (paid conversion)
- Free-tier limits I might hit (database rows, emails/month, hosting bandwidth)
- How to structure the app so adding payments after 6 months is easy

5. **Resources** — Prioritized learning materials for the recommended stack
6. **Budget Forecast** — $0 phase breakdown + projected costs after free period
   (hosting, SMS, payment gateway fees)
7. **Competitor Table** — 2merkato and others: features, pricing, weaknesses
